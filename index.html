Lab AWS EC2 & GitHub

Un guide complet pour maîtriser GitHub et EC2 sur AWS.

Étapes du Lab

  • 1. Créer un compte GitHub :

     

    Rendez-vous sur GitHub et créez un compte si vous n'en avez pas déjà un.

  • 2. Créer un référentiel (repository) :

     

    Créez un nouveau référentiel en utilisant l'interface web de GitHub.

  • 3. Cloner le référentiel :

     

    Utilisez git clone pour cloner le référentiel localement.

  • 4. Pousser des modifications :

     

    Ajoutez des fichiers avec git add ., validez les modifications avec git commit, puis poussez-les sur GitHub avec git push.

  • 5. Configurer IAM sur AWS :

     

    Créez un utilisateur IAM avec les autorisations nécessaires pour interagir avec EC2.

  • 6. Lancer une instance EC2 :

     

    Configurez une nouvelle instance EC2 via la console AWS.

  • 7. Se connecter à EC2 :

     

    Connectez-vous à l'instance EC2 en utilisant SSH et la clé privée téléchargée.

  • 8. Configurer Git sur EC2 :

     

    Installez Git et configurez vos informations utilisateur avec git config.

  • 9. Tirer et vérifier le code :

     

    Utilisez git pull pour récupérer les dernières modifications du référentiel.

  • 10. Terminer et nettoyer :

     

    Supprimez toutes les ressources AWS pour éviter des frais inutiles.
